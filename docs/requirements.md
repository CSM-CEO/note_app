# Техническое задание: Веб-приложение "Заметки с тегами" (Note App)

## 1. Проблема и целевая аудитория
* **Проблема:** У пользователей часто скапливается много разрозненной текстовой информации. Обычные текстовые файлы неудобны, так как в них сложно быстро найти нужную информацию или сгруппировать записи по темам.
* **Решение:** Приложение позволяет быстро создавать заметки, присваивать им один или несколько тегов, фильтровать список по конкретным тегам и искать нужную запись по ключевым словам.
* **Целевая аудитория:** Студенты, разработчики, менеджеры и любые пользователи, которым необходим простой инструмент для организации своих текстовых записей.

## 2. Функциональные требования
1. **Создание заметки:** Пользователь может добавить новую заметку, указав заголовок, текст и теги (через запятую).
2. **Отображение списка:** Главная страница выводит список всех созданных заметок с указанием даты создания и привязанных к ним тегов.
3. **Фильтрация по тегам:** При клике на любой тег приложение фильтрует список и показывает только те заметки, которые содержат данный тег.
4. **Полнотекстовый поиск:** Наличие поисковой строки для фильтрации заметок по совпадению в заголовке или тексте.
5. **Удаление заметок:** Возможность удалить ненужную заметку с автоматическим удалением связей с тегами.

## 3. Архитектура данных (Схема БД)
Для реализации связи "многие-ко-многим" (у одной заметки может быть много тегов, и один тег может принадлежать многим заметкам) используются три таблицы:

1. **notes (Заметки):** `id` (INT, PK), `title` (TEXT), `content` (TEXT), `created_at` (TEXT)
2. **tags (Теги):** `id` (INT, PK), `name` (TEXT, UNIQUE)
3. **note_tags (Таблица связей):** `note_id` (INT, FK), `tag_id` (INT, FK)
