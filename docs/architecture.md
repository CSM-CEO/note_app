# Описание архитектуры веб-приложения "Заметки с тегами"

## 1. Стек технологий и обоснование
* **Язык программирования:** Python 3 — обеспечивает высокую скорость разработки прототипа.
* **Веб-фреймворк:** Flask — легковесный микрофреймворк, идеальный для небольших монолитных приложений.
* **База данных:** SQLite — встроенная реляционная СУБД, не требующая отдельного сервера, что упрощает развертывание прототипа.
* **Фронтенд:** Bootstrap 5 (CSS) + Bootstrap Icons — адаптивный дизайн и готовые компоненты без необходимости писать сложный пользовательский CSS.

## 2. Структура компонентов (Маршруты)
Приложение реализует паттерн клиент-серверной архитектуры со следующими эндпоинтами:
* `GET /` — Главная страница. Отвечает за рендеринг интерфейса, полнотекстовый поиск по ключевым словам и фильтрацию заметок по выбранному тегу.
* `POST /add` — Обработка формы создания новой заметки. Разбирает строку тегов, разделенную запятыми, и сохраняет данные в БД.
* `GET /delete/<id>` — Удаление заметки по её уникальному идентификатору.

## 3. Модель данных (Реляционные связи)
Для реализации полноценной связи "многие-ко-многим" (Many-to-Many) между заметками и тегами спроектированы три таблицы:
* `notes` — хранит заголовки, текст и дату создания заметок.
* `tags` — хранит уникальные имена тегов в нижнем регистре.
* `note_tags` — связующая таблица (содержит внешние ключи `note_id` и `tag_id`), обеспечивающая каскадное удаление связей при удалении заметки.
